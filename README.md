# 왕립 계승 아카데미 봇

마스토돈 기반의 프린스/프린세스 육성 커뮤니티 자동봇입니다.
플레이어는 매 턴 행동을 선택해 수치를 성장시키고, 상가에서 아이템을 구매하거나 무사수행으로 골드를 획득하며 최종 엔딩에 도달합니다.

---

## 봇 구성

봇은 3개의 독립 계정으로 운영됩니다. 각각 별도 토큰이 필요하며 동시에 실행됩니다.

| 봇 | 역할 | 실행 명령 |
|----|------|---------|
| 스케줄 봇 | 턴 행동 제출 / 수치 자동 처리 | `npm run schedule` |
| 상가 봇 | 아이템 구매·판매·장착 / 레스토랑 / 타로 | `npm run shop` |
| 무사수행 봇 | 던전 탐험 (DM) / 레이드·결투 (퍼블릭) | `npm run combat` |

---

## 데이터 구조

플레이어 수치·골드·인벤토리는 Google Sheets에서 실시간으로 읽고 씁니다.
행동 효과, 아이템 정의, 마물 데이터는 시트에서 직접 편집할 수 있어 코드 수정 없이 밸런스 조정이 가능합니다.
전투 세션과 턴 히스토리는 로컬 JSON 파일에 저장됩니다.

## 마스토돈 액세스 토큰 발급

봇 계정 3개를 각각 생성한 뒤, 각 계정으로 로그인해서 아래 과정을 반복합니다.

1. 설정 → 개발 → 새 애플리케이션
2. 권한: `read` `write` `follow` 체크
3. 발급된 토큰을 `.env`의 해당 항목에 입력

GM 계정 ID 확인:
```
GET https://your.instance.social/api/v1/accounts/lookup?acct=GM계정명
```
응답의 `"id"` 값을 `GM_ACCOUNT_ID`에 입력합니다.

## Google Sheets 연동

### 서비스 계정 생성

1. [Google Cloud Console](https://console.cloud.google.com) 접속
2. 프로젝트 생성 또는 선택
3. API 및 서비스 → Google Sheets API 활성화
4. IAM 및 관리자 → 서비스 계정 → 새 서비스 계정 생성
5. 키 탭 → JSON 키 생성 → 다운로드
6. 다운로드한 파일을 `google-key.json`으로 이름 변경 후 프로젝트 루트에 위치

### 스프레드시트 공유

1. Google Sheets에서 새 스프레드시트 생성
2. `google-key.json` 안의 `client_email` 값을 복사
3. 스프레드시트 공유 → 해당 이메일 추가 → 편집자 권한 부여
4. 스프레드시트 URL에서 ID 복사 → `SPREADSHEET_ID`에 입력

URL 예시: `https://docs.google.com/spreadsheets/d/여기가ID/edit`

## 시트 탭 구성

스프레드시트에 아래 5개 탭을 생성하고 1행에 헤더를 입력합니다.

### Players
```
accountId | name | 지능 | 매력 | 체력 | 감성 | 사회성 | 도덕성 | 야망 | 위험도 | 의존성 | 스트레스 | 평판 | 전투 | 골드 | 턴 | 인벤토리 | 장착
```
인벤토리와 장착 칸은 JSON 문자열로 자동 저장됩니다. 직접 편집하지 마세요.

### Actions
```
행동명 | 카테고리 | 최소나이 | 골드 | 지능 | 매력 | 체력 | 감성 | 사회성 | 도덕성 | 야망 | 위험도 | 의존성 | 스트레스 | 평판 | 전투
```
- 카테고리: `교육` `아르바이트` `휴식` `사교` `무사수행`
- 골드: 양수 = 수입, 음수 = 지출
- 비어있는 수치 칸은 0으로 처리됩니다
- 캐시 주기: 5분 (즉시 반영하려면 봇 재시작)

### Items
```
아이템명 | 상점 | 가격 | 판매비율 | 슬롯 | 최소나이 | 지능 | 매력 | 체력 | 감성 | 사회성 | 전투 | 평판 | 스트레스 | 위험도 | 도덕성 | 설명
```
- 상점: `무기상` `의상실` `잡화점` `레스토랑`
- 슬롯 (드롭다운 설정 권장):

| 슬롯 값 | 설명 |
|--------|------|
| `무기` | 무기 슬롯, 장착형 |
| `의상` | 의상 슬롯, 장착형 |
| `악세서리` | 악세서리 슬롯, 장착형 |
| `방패` | 방패 슬롯, 장착형 |
| `소비` | 인벤토리 보관, [사용/이름]으로 소비 |
| `음식` | 레스토랑 전용, 즉시 효과 (인벤토리 미저장) |

- 판매비율: 0.5 = 구매가의 50% / 음식은 0으로 설정
- 캐시 주기: 5분

### Monsters
```
마물명 | 장소 | 종류 | 최소나이 | HP | 공격력 | 방어력 | 골드최소 | 골드최대 | 대화텍스트 | 설명
```
- 종류 (드롭다운 설정 권장): `monster` `boss` `villager`
- 같은 장소에 여러 마물/주민/보스를 등록할 수 있습니다
- 보스는 탐험 마지막 이벤트로 고정 등장합니다
- 캐시 주기: 5분

## 실행

봇을 각각 별도의 터미널에서 실행합니다.

```bash
npm run schedule   # 스케줄 봇
npm run shop       # 상가 봇
npm run combat     # 무사수행 봇
```

개발 중 자동 재시작:

```bash
npm run dev:schedule
npm run dev:shop
npm run dev:combat
```

## 커뮤니티 진행 구조

```
1~8턴    초기 성장기 (8~12세)
          수확제 1회차 (8턴 종료 후 GM 진행)

9~16턴   확장 단계 (13~16세)
          수확제 2회차 (16턴 종료 후 GM 진행)

17~24턴  완성 단계 (17~18세)
          위기 발생 → 레이드 → 졸업식 / 엔딩 판정
```

나잇대가 바뀌면 교육·아르바이트 목록이 자동으로 갱신됩니다.

## 명령어 목록

### 스케줄 봇

**플레이어**

| 명령 | 설명 |
|------|------|
| `[스케줄/행동명] [스케줄/행동명] [스케줄/행동명]` | 이번 턴 행동 3개 제출, 유효하면 즉시 자동 처리 |
| `[스케줄/무사수행/(지역명)] [스케줄/...] [스케줄/...]` | 무사수행 포함 시 지역 지정 필수 |
| `[상태]` | 내 수치를 단어 조합으로 확인 |

**GM**

| 명령 | 설명 |
|------|------|
| `[현황]` | 전체 플레이어 진행 상태 |
| `[상세]` | 전원 숨김 수치 포함 상세 |
| `[상세/이름]` | 특정 플레이어 상세 |
| `[강제진행]` | 미제출 플레이어를 강제로 다음 턴으로 진행 |

**행동 규칙**

- 턴당 동일 행동 2회 이상 선택 시 효율 50% 감소
- 무사수행은 턴당 1회만 선택 가능
- 교육은 골드 지출, 아르바이트는 골드 수입

| 명령 | 설명 |
|------|------|
| `[상가/무기상]` | 무기 목록 출력 |
| `[상가/의상실]` | 의상 목록 출력 (12종) |
| `[상가/잡화점]` | 잡화·장착구·소비 아이템 목록 출력 |

**거래 / 사용**

| 명령 | 설명 |
|------|------|
| `[구매/아이템명]` | 골드 차감 후 인벤토리 추가. 음식은 즉시 효과 |

| `[장착/아이템명]` | 장착 슬롯에 적용. 같은 슬롯 기존 아이템 자동 해제 |

| `[사용/아이템명]` | 소비 아이템(물약·붕대 등) 사용. 인벤토리에서 제거 |
| `[주머니]` | 소지금·장착 중·소지품 확인 |

**타로**

|------|------|

**GM**

|------|------|

### 무사수행 봇

**던전 탐험 (DM)**

스케줄에서 `[스케줄/무사수행/(지역명)]`을 선택한 턴에 봇이 자동으로 DM을 발신합니다.
이후 모든 탐험 진행은 DM에서만 이루어집니다.

탐험은 5개의 이벤트로 구성되며 마지막은 보스 이벤트로 고정됩니다.

| 이벤트 | 선택지 |
|--------|--------|
| 보물상자 | `[열기]` `[지나간다.]` |
| 주민 | `[대화]` `[공격]` `[숨기]` |
| 몬스터 | `[공격]` `[대화]` `[숨기]` |
| 보스 | `[공격]` `[대화]` `[숨기]` |
| 전투 중 | `[공격]` `[도망]` |

선택 결과:

| 선택 | 결과 |
|------|------|
| `[열기]` | 골드 또는 아이템 획득 |
| `[지나간다.]` | 보물상자 무시 후 다음 이벤트 |
| `[대화]` | 주민: 소량 골드 획득 가능. 몬스터/보스: 40% 회피 성공, 실패 시 기습 |
| `[공격]` | 주민: 골드 탈취·도덕성 하락. 몬스터/보스: 전투 시작 |
| `[숨기]` | 65% 성공 시 이탈. 실패 시 전투 |
| `[도망]` | 55% 성공 시 이탈. 실패 시 피해 |

체력이 0이 되면 아카데미로 강제 귀환되며 소지금의 10%를 잃습니다.
귀환 후에는 무사수행 봇 DM에서 어떤 키워드를 입력해도 응답하지 않습니다.

**퍼블릭 레이드**

| 명령 | 설명 |
|------|------|
| `[레이드/보스명]` | 레이드 개설 (GM 전용) |
| `[참가]` | 레이드 참가 신청 |
| `[레이드시작]` | 전투 시작 (GM 전용) |
| `[공격]` | 보스 공격 |
| `[레이드종료]` | 강제 종료 (GM 전용) |

레이드 승리 시 참가자 전원에게 보상 골드를 균등 분배하며 피해량 순위를 공개 게시합니다.

**퍼블릭 1:1 결투**

| 명령 | 설명 |
|------|------|
| `[결투/상대계정]` | 결투 신청 |
| `[수락]` | 결투 수락 |
| `[공격]` | 교대로 공격 |

## 수치 시스템

**공개 수치** — 플레이어에게 단어 조합으로만 표시됩니다.

| 수치 | 낮음 (0~15) | 중간 (30~50) | 높음 (85~100) |
|------|------------|-------------|--------------|
| 지능 | 무지한 | 총명한 | 천재적인 |
| 매력 | 눈에 띄지 않는 | 친근한 | 전설적인 |
| 체력 | 허약한 | 건강한 | 불굴의 |
| 감성 | 무감각한 | 섬세한 | 예술적인 |
| 사회성 | 고독한 | 사교적인 | 카리스마 넘치는 |

**숨겨진 수치** — 플레이어에게 비공개이며 GM만 `[상세]` 명령으로 확인할 수 있습니다.

`도덕성` `야망` `위험도` `의존성` `스트레스` `평판` `전투`

## 주의사항

- `google-key.json`과 `.env`는 절대 깃에 올리지 마세요. `.gitignore`에 포함되어 있습니다.
- `data/` 폴더도 커밋에서 제외됩니다. 서버 이전 시 별도 백업이 필요합니다.
- Actions, Items, Monsters 시트 데이터는 5분 간격으로 캐시됩니다. 수정 후 즉시 반영하려면 봇을 재시작하세요.
- Items 시트의 슬롯 열은 드롭다운으로 설정하고 값이 정확히 일치해야 합니다. (무기 / 의상 / 악세서리 / 방패 / 소비 / 음식)
- 봇 3개는 각각 다른 마스토돈 계정을 사용해야 합니다.
